Hazme una aplicación para descargar videos de cualquier plataforma como por ejemplo YouTube similar a la pagina y2mate, usando spring boot, java en el backend, y bootstrap5 para el frontend, y yt-dlp y ffmpeg para la obtención de metadatos, descarga y obtención de mejor calidad de video y audio.

Frontend:
Utiliza bootstrap5 para hacer el frontend, y una UI idéntica a las imágenes de referencia, tiene que haber tres secciones, la de video con tipos de archivo de video(con las 6 o menos primeras calidades de video mp4, que estén entre 140p a 1100p que se obtendrán de los metadatos), la de audio con formatos de audio(320 kbps, 240kb), y la de other con archivos distintos, como webm.

Backend:
Utiliza spring boot y java para llevar a cabo el backend del proyecto, se utilizara la librería yt-dlp para realizar la tarea de descargar los videos, y se utilizara ffmpeg para obtener una mejor calidad, de video, audio, webm.

funcionamiento:
Se pegara el enlace en la textfield, y al darle al botón "start"  aparecerá una barra de progresión que reflejara el porcentaje de obtención de los metadatos o datos necesarios que contendrán, la miniatura del video, titulo, formatos video, audio, webv disponibles y los necesarios para descargar el video, audio, o webv. Al completarse la barra de progresión, dichos datos se pasaran y se mostraran en el frontend como en la imagen de referencia, que serán tres secciones, la de video, la de audio, y la de other con sus respectivos formatos como ya se había especificado y se referencia en las imágenes de referencia, al seleccionar el formato deseado, por ejemplo en la sección de video se selecciona descargar una calidad de 720p por ejemplo, el backend debera ejecutar el comando en yt-dlp y ffmpeg correspondiente para obtener dicha calidad, a continuación se mostrara un barra de progresión real que estará obteniendo los datos necesarios para descargar el video, audio, o webv en el formato y calidad seleccionada, al completarse la barra de progresión, aparecerá un botón de descarga, el cual al pulsarlo, el navegador procederá a descargar el video, audio, webm. La UI/UX debe verse como en la imagen de referencia, y si una URL no esta disponible ya sea porque el video no es publico u otros errores, debe aparecer un label que dice que la url no esta disponible, se mostraran solo los formatos de resolucion disponibles del archivo, y si no hay formatos de resolucion disponibles solo no se muesstran.
